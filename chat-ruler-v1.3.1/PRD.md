# 对话标尺 Chat Ruler - 产品需求文档（PRD）

## 1. 需求评估结论

### 1.1 需求合理性

**合理且高价值。** 理由：

1. **痛点真实存在**：在与AI进行多轮深度对话（尤其是编程、写作、分析类任务）时，对话内容往往非常长。当AI回复了数千字后，用户想找回之前某一轮的上下文极为困难，只能靠鼠标滚轮反复翻找。
2. **交互模式成熟**：文档大纲导航（TOC Sidebar）是钉钉文档、飞书文档、Notion、Typora 等产品的标配交互，用户学习成本极低。将这种成熟的"文档内导航"模式迁移到"对话内导航"是自然延伸。
3. **竞品验证**：市面上已有 Chat2PDF、NousSave、AI Outline Companion 等类似插件获得成功，说明用户需求确实存在。
4. **技术可行性强**：纯前端内容脚本（Content Script）即可实现，无需调用AI API，无需后端服务，零隐私风险。

### 1.2 风险与边界

| 风险点 | 影响 | 缓解措施 |
|--------|------|----------|
| AI平台DOM变更 | 功能失效 | 模块化适配器，快速更新选择器 |
| 侧边栏与页面原有UI冲突 | 视觉遮挡 | 提供显隐开关、宽度可调、位置可选 |
| 超长对话性能问题 | 卡顿 | 虚拟滚动/分页渲染、防抖更新 |
| 平台反爬虫机制 | 脚本被拦截 | 仅读取DOM，不修改页面原有逻辑 |

## 2. 产品概述

### 2.1 产品名称
**对话标尺 | Chat Ruler**

### 2.2 产品定位
为重度AI用户在多轮长对话中提供"轮次级"导航能力，让用户像浏览文档大纲一样快速定位和跳转对话历史。

### 2.3 目标用户
- 每天与AI进行10轮以上深度对话的重度用户
- 使用AI进行长文写作、代码审查、数据分析等复杂任务的用户
- 需要反复回顾AI历史回复内容的用户

## 3. 功能需求

### 3.1 核心功能（P0）

#### 3.1.1 对话轮次自动识别
- 自动将连续的对话流划分为"轮次"（Turn/Round）
- 一轮 = 用户提问（User）+ AI回复（Assistant）
- 如果用户连续发送多条消息，合并为一轮
- 如果AI回复包含多条消息，合并为一轮

#### 3.1.2 侧边栏目录导航
- 在页面右侧注入可折叠的侧边栏面板
- 面板上显示所有对话轮次的索引列表
- 每个轮次显示：轮次编号 + 用户提问摘要
- 点击任意轮次，页面瞬跳并高亮目标消息
- 跳转后，目标消息边框高亮闪烁提示（1.5秒后消失）

#### 3.1.3 当前位置跟随高亮
- 监听页面滚动（IntersectionObserver）
- 当前视口内最显著的轮次，在侧边栏对应项高亮显示
- 高亮样式：左侧边框 + 背景色变化

#### 3.1.4 跨平台适配
支持以下7个平台的网页版：
1. **Kimi** (kimi.moonshot.cn / kimi.com)
2. **Gemini** (gemini.google.com)
3. **DeepSeek** (chat.deepseek.com)
4. **豆包** (doubao.com)
5. **元宝** (yuanbao.tencent.com)
6. **ChatGPT** (chatgpt.com / chat.openai.com)
7. **Sider** (sider.ai)

#### 3.1.5 自动更新与监听
- 使用 MutationObserver 监听对话容器变化
- 新消息到达时自动刷新侧边栏
- 防抖处理：DOM变化密集时，延迟300ms后统一刷新

### 3.2 增强功能（P1）

#### 3.2.1 用户输入摘要
- 在侧边栏展示用户输入内容的前14个字
- 超过14字截断并加省略号
- 优先提取纯文本，过滤掉代码块、图片等富媒体标记

#### 3.2.2 AI回复标题展示
- 解析AI回复内容中的 Markdown 一级/二级标题
- 在对应轮次下方以子项形式展示这些标题
- 点击标题可跳转到对应消息内的具体位置
- 支持展开/收起标题列表

### 3.3 体验优化（P2）

#### 3.3.1 侧边栏交互
- 拖拽调整侧边栏宽度（最小200px，最大500px）
- 记忆用户的宽度偏好（localStorage）
- 快捷键 `Alt+Shift+O` 快速显隐侧边栏
- 面板折叠为迷你导航条，hover 时抽屉展开，坐标对齐不跳动

#### 3.3.2 视觉融合
- 侧边栏统一使用 `#2563EB` 蓝色主题
- 圆角、阴影、过渡动画采用 Apple 风格
- 使用 CSS backdrop-filter 实现毛玻璃效果
- 暗色/亮色模式自适应

## 4. 非功能需求

### 4.1 性能
- 打包体积 < 200KB（无第三方依赖）
- 首次渲染 < 100ms（50轮以内对话）
- 滚动跟随时无肉眼可见卡顿

### 4.2 兼容性
- Chrome 110+（Manifest V3）
- Edge、Brave 等 Chromium 内核浏览器
- 仅注入目标域名，其他域名完全静默

### 4.3 安全隐私
- 零网络请求（无远程API调用、无统计埋点）
- 所有处理在本地浏览器完成
- 不采集、不上传任何对话内容
- 最小权限原则（仅需 activeTab + storage）

### 4.4 可维护性
- 模块化平台适配器，新增平台仅需扩展配置对象
- 选择器与逻辑分离，DOM变更时只需更新选择器
- 版本管理遵循语义化版本

### 5.2 模块说明

| 模块 | 职责 |
|------|------|
| DETECTORS | 每个平台的DOM选择器与消息检测逻辑 |
| DOM Parser | 扫描页面消息元素，按规则配对为"轮次"，提取摘要和标题 |
| Anchor Manager | 为每轮消息注入唯一锚点ID，管理滚动定位 |
| Sidebar Renderer | 构建侧边栏DOM树，绑定点击事件，更新高亮状态 |
| Scroll Watcher | IntersectionObserver监听可视区域，同步侧边栏高亮 |
| Mutation Watcher | MutationObserver监听新消息，触发目录刷新（防抖） |
| Theme Manager | 暗色/亮色模式检测与自适应 |
| Storage Manager | localStorage读写用户偏好（宽度、显隐、折叠状态） |

## 6. 交互设计

### 6.1 侧边栏状态

```
[展开状态]                    [折叠状态 — Mini Bar]
┌────────┬────────────────┐   ┌─┬────────────────┐
│ 收起   │ 对话标尺        │   │1│                │
│ Round 1│ ┌──────────┐   │   │2│   (页面内容)    │
│ ├ 用户 │ │1 怎么使用...│   │   │3│                │
│ │ 提问 │ │2 帮我写代码 │   │   │4│                │
│ Round 2│ │3 优化一下  │   │   │…│                │
│ ├ 用户 │ │   ├ 标题A  │   │   │⇄│                │
│ │ 提问 │ │   ├ 标题B  │   │   └─┴────────────────┘
│ │  ①   │ └──────────┘   │
│ Round 3│                │
│ ...    │                │
└────────┴────────────────┘
```

### 6.2 Mini Bar 抽屉交互
- 未 hover：右侧显示一列数字序号，32×32px 正方形
- hover：宽度从 48px 展开到 240px，序号坐标对齐，不跳动
- 展开后列表：浅蓝色 active 底纹（左右 8px margin，不撑满），右侧保留横杠指示器

### 6.3 跳转反馈
- 点击侧边栏项 → 页面瞬跳至对应消息顶部
- 目标消息边框高亮（蓝色脉冲动画）
- 1.5秒后高亮自动消失

### 6.4 快捷键
| 快捷键 | 功能 |
|--------|------|
| `Alt+Shift+O` | 切换侧边栏显隐 |
| `Alt+Shift+R` | 手动刷新目录 |

## 7. 平台适配详情

| 平台 | 域名 | 消息检测方式 |
|------|------|-------------|

## 8. 发布与安装

### 8.1 安装方式
开发者模式本地加载：
1. 下载插件目录
2. 打开 Chrome `chrome://extensions/`
3. 开启右上角"开发者模式"
4. 点击"加载已解压的扩展程序"
5. 选择项目根目录

### 8.2 后续迭代
- v1.4：新增 Claude、Copilot 适配
- v1.5：支持导出对话标尺为书签/大纲文件

- 安装后用户在长对话（>10轮）中的平均滚动次数减少 50%+

- 各平台消息识别准确率 > 95%
